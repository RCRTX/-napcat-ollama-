# QQ群聊监控系统使用教程

## 适用场景

这个工具用于监听指定QQ群消息，保存聊天记录，并用本地或兼容 OpenAI 接口的模型审查违规内容。发现违规后，系统可以在网页面板展示告警，也可以通过 QQ 私聊、邮箱、Server酱或 PushPlus 通知管理员。

## 准备环境

需要准备：

1. Windows 10/11。
2. Python 3.8 或更高版本。
3. NapCat，并登录用于监听群消息的 QQ。
4. Ollama，或其他 OpenAI 兼容接口。

NapCat 需要开启：

```text
HTTP API: http://localhost:3000
WebSocket: ws://localhost:3001
```

如果你修改了 NapCat 的端口，需要同步修改 `config/config.json` 里的 `napcat.http_url` 和 `napcat.ws_url`。

## 安装步骤

1. 解压分发包。
2. 打开解压后的项目目录。
3. 安装依赖：

```bat
pip install -r requirements.txt
```

4. 复制示例配置：

```bat
copy config\config.example.json config\config.json
```

5. 编辑 `config/config.json`。

## 配置说明

### NapCat

```json
"napcat": {
    "ws_url": "ws://localhost:3001",
    "http_url": "http://localhost:3000",
    "token": "",
    "monitor_groups": [123456789]
}
```

`monitor_groups` 填要监听的群号。可以填多个，例如：

```json
"monitor_groups": [123456789, 987654321]
```

如果 NapCat 配了访问令牌，把令牌填到 `token`；没有就留空。

### AI审查

默认使用 Ollama：

```json
"ai_review": {
    "enabled": true,
    "api_base": "http://localhost:11434/v1",
    "api_key": "ollama",
    "model": "qwen2.5",
    "review_interval_minutes": 5
}
```

先在终端确认模型可用：

```bat
ollama list
```

如果没有模型，可以拉取：

```bat
ollama pull qwen2.5
```

也可以改成其他模型名，例如 `gemma2:9b`。云端或特殊模型需要先确认本地 Ollama 已完成授权。

### QQ私聊通知

如果想在发现违规后收到 QQ 私聊通知：

```json
"alert": {
    "qq": {
        "enabled": true,
        "recipient_user_ids": [123456789]
    }
}
```

`recipient_user_ids` 填接收通知的 QQ 号。接收方通常需要是机器人登录账号的好友，或者允许接收该账号私聊。

### 邮箱和推送

邮箱通知需要 SMTP 授权码，不是邮箱登录密码。QQ邮箱需要在邮箱设置里开启 SMTP 服务并生成授权码。

Server酱和 PushPlus 需要在对应平台获取 SendKey 或 Token，再填入配置。

## 启动

确认 NapCat 已经运行后，双击：

```bat
start.bat
```

启动后浏览器访问：

```text
http://localhost:8080
```

页面里可以查看：

- 数据概览
- 聊天记录
- 违规告警
- 群组管理
- AI设置

## 测试通知

可以运行：

```bat
test_alert.bat
```

如果配置了 QQ 私聊通知，接收 QQ 会收到一条测试告警。

## 常见问题

### Web面板显示未连接

检查 NapCat WebSocket 是否开启，配置里的地址是否为：

```text
ws://localhost:3001
```

### QQ私聊通知失败

先确认 NapCat HTTP API 可访问：

```text
http://localhost:3000/get_login_info
```

如果返回登录账号信息，说明 HTTP API 正常。还需要确认接收通知的 QQ 能收到机器人账号的私聊。

### AI审查没有结果

先在 Web 面板的 AI 设置里点击“测试连接”。如果失败，检查 Ollama 是否启动、模型名是否正确。

### 聊天记录保存在哪里

机器可读数据在 `data` 目录；普通人可直接阅读的文本记录在 `聊天记录` 目录。两个目录都在项目解压后的目录内，除非你修改了 `config/config.json`。
